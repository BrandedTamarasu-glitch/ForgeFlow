Forgeflow V4.3.42 · 2 June 2026

Release: Readiness, Wrapper, Learning, And Context Refinements

Forgeflow 4.3.42 improves post-release install readiness, command-wrapper cleanup planning, learning signal trust, context-budget trim guidance, and the first-use path while keeping every new behavior local and advisory.

Release And Wrapper Readiness

  • /forgeflow-release-follow-through now reports install-readiness blockers and informational follow-ups, treating install attention as blocking instead of install-ready.
  • command-wrapper-contract.js now groups issue counts by type and renders ranked next-batch actions in Markdown and JSON.
  • The first wrapper cleanup batch reduced remaining contract drift from 50 issues to 40 while preserving helper fallback behavior.

Learning And Context Guidance

  • /forgeflow-learning-status now identifies trusted sources, weakest sources, and one next quality action before agents rely on local guidance.
  • advise-context.js adds an advisory auto-trim rollup only when trimming is actually recommended, with proof-preserving stop rules.
  • Zero-token budget targets remain stable in JSON instead of serializing as null.

First-Use Path

  • /forgeflow-first-useful-win now prints the first-use path for install health, profile bootstrap, first task reporting, workflow-ending capture, and shareable summary.
  • README and wiki docs were updated for the new release, wrapper, learning, context, and first-use outputs.
  • Command docs now use file-existence checks where helpers are launched through node, avoiding unnecessary repair guidance for readable helper files.

Validation

  • Full scripts/forgeflow/test-*.js suite
  • node scripts/forgeflow/smoke-check.js --mode source --json
  • node scripts/forgeflow/test-release-version.js
  • node scripts/forgeflow/test-doc-links.js
  • git diff --check
  • Forgeflow full-mode review synthesis approved after reviewer findings were fixed
