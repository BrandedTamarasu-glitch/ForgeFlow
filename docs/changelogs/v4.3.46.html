Forgeflow V4.3.46 · 2 June 2026

Release: Wrapper Drift Closure And Decision Briefs

Forgeflow 4.3.46 closes the safe command-wrapper drift backlog and adds read-only helpers for wrapper planning, release consumption follow-through, learning capture nudges, and project decision briefs.

Command Wrapper Safety

  • Cleared every low-risk missing-node-env-scrub wrapper-contract issue by routing helper calls through env-scrubbed Node execution.
  • Added missing repair guidance to /forgeflow-support.
  • Added /forgeflow-wrapper-drift-plan to separate safe mechanical cleanup from manual or high-risk wrapper work.
  • The only remaining wrapper-contract item is the intentionally deferred high-risk commands/review.md safe-args change.

Follow-Through Helpers

  • Added /forgeflow-release-consumption-loop to show the next post-release update, smoke, or consumption step without running updates, repairs, smoke checks, or snapshot writes.
  • Added /forgeflow-learning-capture-nudge to show the exact capture command after review, next-work, agent-feedback, or first-run events without inventing observed values.
  • Added /forgeflow-project-brief to summarize existing project learnings, latest insights, code topology, and health timeline evidence into a concise next-work decision brief.

Managed Install And Docs Wiring

  • Wired the new commands and runtime helpers through health, install manifest, update tests, release checks, README, and wiki pages.
  • The project brief now reads real topology counts from nested topology summaries and derives health timeline state from the existing health-timeline helper.
