Forgeflow V4.3.50 · 3 June 2026

Release: Workflow Automation Handoffs

Forgeflow 4.3.50 improves the safe automation path between readiness, review waves, evidence capture, profile setup, telemetry quality, and runtime inventory.

Context And Review Handoffs

  • /forgeflow-context-wave-build now reports an automation handoff with write boundary, verification command, next command, and focused review-packet details.
  • Workflow readiness now surfaces runtime coordination-pressure guidance while preserving read-only behavior.
  • The review-wave path remains explicit: it does not spawn reviewers, trim proof files, edit source files, commit, or push.

Evidence And Profile Guidance

  • /forgeflow-workflow-ending-capture now includes an evidence contract with required values and do-not-record boundaries for review, next-work, and agent-feedback events.
  • /forgeflow-profile-bootstrap now separates required operating prompts from optional workflow and optional project-style prompts.
  • /forgeflow-telemetry-quality now includes an evidence ladder and only marks calibration usable when the stronger routing threshold is met.

Runtime Coordination

  • runtime-inventory.js now reports read-only coordination pressure so maintainers can see registry hot files without changing install/update behavior.
  • README and wiki docs now describe the new handoff, evidence, prompt-group, telemetry, and runtime-inventory surfaces.
  • High-risk commands/review.md safe-args work remains paused and untouched.
