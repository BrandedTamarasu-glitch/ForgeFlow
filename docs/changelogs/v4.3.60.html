Forgeflow V4.3.60 · 8 June 2026

Release: Review-Auto Status And Recovery View

Forgeflow 4.3.60 lands Phase 7 of the review-auto sandbox path. The deterministic proposal flow now has a read-only status surface that shows what exists, what failed, what was applied, and the next safe action.

Status Command

  • New /forgeflow-review-autofix-status summarizes proposal inputs, sandbox proposal artifacts, apply artifacts, apply history, invalid artifacts, failures, and rollbacks.
  • New show-review-autofix-status.js supports Markdown and JSON output for local automation and human review.
  • The helper reports a concrete next action, such as sandboxing a proposal input, applying a validated sandbox artifact, inspecting a failed artifact, or running review after a successful apply.

Recovery Signals

  • Validation-failed sandbox proposals and validation-failed rolled-back apply artifacts are surfaced as attention states with artifact paths for inspection.
  • Invalid JSON, symlinked artifacts, non-regular files, and unsafe project artifact paths are counted and reported without following unsafe filesystem entries.
  • Attention states remain informational command output, not shell-level command failures.

Runtime And Docs

  • Install, update, health, runtime contract, release-check, release-gate, README, and wiki references now include the new status helper and command.
  • Command coverage now tracks 94 Forgeflow commands.
  • Release readiness now checks 121 managed runtime helper sources.

Safety Boundaries

  • The status command is read-only.
  • It does not edit files, generate proposals, apply proposals, run validation, commit, push, call GitHub, dispatch workers, batch fixes, or claim review approval.
