Forgeflow V4.3.1 · 21 May 2026

Patch Release: Downstream Smoke Fixes And Review Hardening

Forgeflow 4.3.1 is a patch release for the first public GitHub issue plus the review, ship, update, install, and local-context hardening completed after 4.3.0.

Downstream Smoke Fix

  • Fixed smoke-check.js so installed downstream projects no longer fail doc-links and release-version when source-tree test files are not installed.
  • The smoke helper now runs those checks when the Forgeflow source test scripts are available and reports skip with an explicit reason when they are not.
  • Updated /forgeflow-smoke, README, and wiki docs to describe doc/release guards as optional source-tree checks.

Review And Ship Hardening

  • /review records final verdicts to review history so /ship can use a concrete approval handoff.
  • /review-auto refreshes project learnings after auto-fix rounds, records post-fix approval state, and uses path-scoped rollback guidance.
  • /ship treats potential secrets as hard blockers and enforces commit hygiene without depending on local project opt-in files.

Install, Update, And Context Safety

  • Install/update helpers reject symlinked managed sources, account for deleted managed files, and report repair-needed installs when required runtime files are missing.
  • Context, memory, scope, and learning helpers reject symlinked local artifact paths where they read or write Forgeflow state.
  • Context packs and scope manifests include untracked files so newly created work appears in generated review and implementation context.

Validation

  • Full release-gate helper tests passed locally before tagging.
  • Source-tree smoke still runs doc/release guards and passes.
  • Downstream optional smoke guard behavior was verified with a consumer-style temp root.
