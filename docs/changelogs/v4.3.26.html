Forgeflow V4.3.26 · 28 May 2026

Release: Local Intelligence Refinement Batch

Forgeflow 4.3.26 turns the recent learning and runtime checks into clearer operational guidance. It adds install consumability evidence, signal-quality scoring, per-agent context contracts, runtime repair previews, and a project health timeline.

Release And Runtime Confidence

  • /forgeflow-release-verify now includes installed-version and runtime-drift consumability evidence in the post-publish summary.
  • /forgeflow-runtime-drift --preview-repair shows the exact managed helper actions that /update-forgeflow --repair would take, without mutating installed files.
  • Installed-runtime dogfood covers the new timeline and release verification behavior.

Learning And Agent Context

  • /forgeflow-learning-status now scores local learning signals as high, medium, or low trust, with notes for sparse, stale, issue-heavy, or corrective-heavy signals.
  • Context packs now write agent-context-contract.json and include a role-specific Agent Context Contract in each generated packet.
  • The contract states which signals are allowed, advisory, verify-before-use, and prohibited so local learning never becomes unverified authority.

Project Health Timeline

  • New /forgeflow-health-timeline renders a read-only local timeline across code-map history, context-advisor history, latest-insights readiness, and learning-signal quality.
  • README and wiki docs now cover the timeline, signal-quality scoring, context contracts, runtime repair preview, and install consumability checks.
  • The new timeline stays advisory and does not refresh artifacts, approve work, call the network, or mutate project files.

Validation

  • Full scripts/forgeflow/test-*.js suite
  • node scripts/forgeflow/smoke-check.js --mode source --json
  • git diff --check
  • Focused release verify, runtime drift, learning status, health timeline, context pack, command coverage, install/update, docs, and installed-runtime dogfood tests
