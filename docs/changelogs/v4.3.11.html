Forgeflow V4.3.11 · 26 May 2026

Release: Project Intelligence And Adoption Decisions

Forgeflow 4.3.11 packages the net-new adoption track: context packs are easier to run from target checkouts, source smoke explains gate outcomes, project intelligence summarizes local guidance for review prep, and adoption packs now produce public-safe decision handoffs.

Context And Smoke

  • build-context-pack.js --root <repo> supports explicit target roots for cross-checkout review preparation.
  • Source smoke now includes installed-runtime dogfood coverage so packaged helper drift is caught before release.
  • Smoke and downstream gates report clearer pass, warn, and fail reasons with next actions.

Project Intelligence

  • build-project-intelligence.js writes a compact review-prep rollup with trust state, freshness, Git provenance, top risks, hot files, validation patterns, and artifact pointers.
  • Agent feedback is consumed as advisory-only signal with confidence/evidence metadata, by-agent counts, skipped-line reasons, and schema/privacy revalidation before rollup.
  • Project-learning gate failures no longer hide unrelated freshness, import-gap, failure-digest, or context-advisor risks.

Adoption Pack

  • render-adoption-pack.js now includes a public-safe summary, fixed decision vocabulary, small-team handoff checklist, and fill-in decision template.
  • Public-safe summaries avoid raw project identity, free-form support categories, private notes, and private host/path strings.
  • Generated handoff commands quote project names safely and preserve Claude Code and Codex runtime paths.

Validation

  • Full local release-gate validation passed before tagging.
  • Aegis verified the high-risk privacy and command-safety fixes for project intelligence and adoption summaries.
  • Arbiter and Compass approved the learning and adoption slices before release.
