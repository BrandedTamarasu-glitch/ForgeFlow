Forgeflow V4.3.18 · 27 May 2026

Release: Issue-Aware Releases And Local Code-Map Acceptances

Forgeflow 4.3.18 improves the release loop and downstream smoke clarity. Release-note drafts can surface referenced issues safely, smoke explains expected import gaps more clearly, code-map can read local exact-gap acceptances, and release readiness can run a local post-publish verification pass.

Release Loop

  • render-release-notes.js now extracts local #123 issue references from recent commit subjects and renders an advisory referenced-issues section.
  • Release docs now require verifying issue state before claiming an issue is fixed or closed.
  • /forgeflow-release-readiness --post-publish adds local advisory verification for plugin version, local tag, changelog, release-note draft, source smoke, and update smoke evidence.
  • Post-publish verification supports patch-zero changelog filenames such as v4.2.html.

Code Map And Smoke

  • /forgeflow-smoke now explains production, expected, local-accepted, and needs-review import-gap counts separately.
  • /forgeflow-code-map can read local-only exact acceptances from .forgeflow/<project>/code-map-accept.json.
  • Accepted gaps stay visible as accepted-local, count as expected, and do not hide stale or invalid acceptance entries.
  • Malformed acceptance files now report invalid shape instead of being silently ignored.
