Forgeflow V4.3.44 · 2 June 2026

Release: Action Routing, First-Run Simulation, And Safer Follow-Through

Forgeflow 4.3.44 makes local guidance more directly usable. Context advice now points to real next commands, learning quality can route weak evidence streams to runnable follow-up checks, release follow-through can save a protected local snapshot, and fresh-user trials get a read-only first-run simulator.

Actionable Local Guidance

  • /forgeflow-context-advisor now includes next_actions and review_wave_suggested so budget or low-savings findings point to the next real helper or review-wave command.
  • /forgeflow-learning-action ranks weak learning and telemetry sources, then emits one runnable local follow-up command.
  • First-run learning gaps route to /forgeflow-first-run-rollup rather than an incomplete recording command.

First-Run And Release Evidence

  • /forgeflow-first-run-simulator checks plugin version evidence, runtime-specific first-use steps, and source-smoke readiness without installing, repairing, updating, or writing evidence.
  • The simulator reads the actual Claude plugin manifest first, with Codex plugin metadata as a fallback for tests and future packaging paths.
  • /forgeflow-release-follow-through --save can persist the latest local release-consumption snapshot, using Forgeflow's safe JSON writer to reject symlinked or unsafe destinations.

Wrapper And Install Safety

  • The next command-wrapper cleanup batch reduced baseline wrapper drift from 32 issues to 27 while keeping the remaining drift visible for future focused slices.
  • New command wrappers are covered by smoke checks for helper fallback, regular-file probes, safe argument forwarding, runtime validation, unsupported argument rejection, and Node environment scrubbing.
  • Health, install, update, README, and wiki surfaces now include the new first-run simulator and learning action command.
