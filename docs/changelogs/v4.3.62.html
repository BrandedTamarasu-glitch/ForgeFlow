Forgeflow V4.3.62 · 8 June 2026

Release: Ownership Recommendation Map

Forgeflow 4.3.62 lands Phase 9 of the architecture intelligence path. Forgeflow can now render a local advisory ownership map that connects topology hotspots, operating-model high-care files, architecture evidence, runtime owner surfaces, and optional CODEOWNERS coverage.

Ownership Command

  • New /forgeflow-ownership renders owner-surface recommendations for review routing and high-care files.
  • New render-ownership-map.js supports Markdown and JSON output for local review and automation.
  • Default output is read-only; --write stores local .forgeflow/<project>/context/ownership-map.md and ownership-map.json only.

Evidence And Coverage

  • The report groups files into owner surfaces such as context intelligence, install/update/health, command wrappers, release shipping, learning evidence, user profile, hooks, and runtime core.
  • Optional CODEOWNERS files are parsed conservatively from CODEOWNERS, .github/CODEOWNERS, or docs/CODEOWNERS.
  • Missing CODEOWNERS is not a failure. Forgeflow still reports recommended lanes, high-care files, and coverage gaps from local evidence.
  • Learned hot-file annotations are normalized so reports show clean paths instead of signal-count suffixes.

Runtime And Docs

  • Install, update, health, runtime inventory, release-check, release-gate, README, and wiki references now include the ownership helper and command.
  • Command coverage now tracks 96 Forgeflow commands.
  • Release readiness now checks 123 managed runtime helper sources.

Safety Boundaries

  • The command does not edit CODEOWNERS, call GitHub, assign reviewers, infer permissions, commit, push, publish, or claim review approval.
  • Symlinked or malformed local evidence is surfaced as attention instead of being followed blindly.
