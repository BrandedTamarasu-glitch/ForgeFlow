Forgeflow V4.3.49 · 3 June 2026

Release: Workflow Follow-Through Runbooks

Forgeflow 4.3.49 strengthens the workflow-readiness path with concrete follow-through commands, visible stop rules, and safer local runbooks for agent-heavy work.

Readiness Follow-Through

  • /forgeflow-review-wave-prep now reports whether review waves need file lists, are ready to build, can use the current packet, or need a rebuilt context pack.
  • /forgeflow-workflow-readiness now carries that follow-through state into the top-level next action, including ready-to-build wave commands.
  • The workflow-readiness markdown now exposes per-step stop rules, observed-evidence requirements, write markers, and automation stop rules.

Learning And Profile Safety

  • /forgeflow-outcome-capture-plan now includes observed-evidence capture runbooks and do-not-record boundaries for each outcome stream.
  • /forgeflow-profile-bootstrap now shows a guided setup path from required operating preferences to preview, confirmed write, and profile check.
  • Profile bootstrap write guidance now prints runnable commands from explicit previewed flags instead of placeholder text.

Release Dogfood

  • /forgeflow-release-consumption-loop now includes a read-only dogfood report that summarizes update, downstream smoke, and release-consumption evidence.
  • README and wiki command docs now describe the follow-through, runbook, profile, and dogfood report behavior.
  • High-risk commands/review.md safe-args work remains paused and untouched.
