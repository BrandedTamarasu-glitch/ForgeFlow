Forgeflow V4.3.63 · 9 June 2026

Release: Runtime Invocation Hints

Forgeflow 4.3.63 lands Phase 10 of the architecture intelligence path. Forgeflow can now render local advisory runtime entrypoint and invocation hints without executing project code or claiming a full runtime call graph.

Invocation Command

  • New /forgeflow-invocation-hints renders package scripts, package entry fields, bins, likely source entrypoints, route-like files, and common framework/test/build config hints.
  • New render-invocation-hints.js supports Markdown and JSON output for local review and automation.
  • Default output is read-only; --write stores local .forgeflow/<project>/context/invocation-hints.md and invocation-hints.json only.

Evidence And Filtering

  • The report reads package metadata, common config files, current code topology, and architecture evidence when present.
  • Nested package scripts include directory-aware run hints such as cd services/api && npm run dev.
  • Dependency folders, build output, test paths, and fixture paths are skipped so the report focuses on real project invocation surfaces.
  • Missing architecture evidence is surfaced as a gap, not a blocker.

Runtime And Docs

  • Install, update, health, runtime inventory, release-check, release-gate, README, and wiki references now include the invocation helper and command.
  • Command coverage now tracks 97 Forgeflow commands.
  • Release readiness now checks 124 managed runtime helper sources.

Safety Boundaries

  • The command does not execute package scripts, start servers, install dependencies, trace runtime behavior, infer a full call graph, commit, push, publish, or claim review approval.
  • Symlinked or malformed local evidence is surfaced as attention instead of being followed blindly.
