Forgeflow V4.3.38 · 29 May 2026

Release: Safer Efficiency Planning And Command Indexing

Forgeflow 4.3.38 tightens the token-efficiency loop with live budget evidence, smarter review waves, safer copy-ready prompts, and a generated command index for thinner docs.

Context And Budget Signals

  • /forgeflow-efficiency-gaps now reads live context-advisor budget state instead of trusting stale rollup evidence when the two disagree.
  • /forgeflow-context-wave-plan flags incomplete zero-file or missing-agent-packet context before review starts.
  • /forgeflow-review-wave-prep now blocks incomplete context with rebuild guidance instead of treating it as usable.

Smarter Review Waves

  • Wave planning ranks files by path risk, topology hubs, changed-neighborhood hints, and proof-file markers.
  • Generated wave commands now shell-quote file-list arguments, including placeholder and written wave-file paths.
  • Proof files remain visible in wave output so budget trimming does not hide validation evidence.

Learning And Command Surfaces

  • /forgeflow-next-work-ranking includes shell-safe outcome-capture prompts for recording whether recommendations were useful, ignored, incorrect, or blocked.
  • runtime-inventory.js exposes shared command/helper summaries, helper groups, installed names, and registry counts.
  • New render-command-index.js generates a compact command index from command frontmatter and runtime-inventory command discovery, including nested command colon names.

Validation

  • Full scripts/forgeflow/test-*.js suite
  • node scripts/forgeflow/smoke-check.js --mode source --json
  • git diff --check
  • Full Forgeflow re-review with Smith, Warden, Lumen, and Atlas approved after fixes
