Forgeflow V4.3.21 · 27 May 2026

Release: Local User Profile Guidance

Forgeflow 4.3.21 adds local user profile guidance so agents can adapt to how a user likes to work while preserving project-specific look, feel, copy, accessibility, and workflow preferences.

User Operating Preferences

  • New /forgeflow-profile command records, checks, and shows local advisory profile guidance.
  • Global operating preferences live under ~/.claude/forgeflow/ and capture collaboration style, autonomy, validation, release, docs, risk, review, and workflow preferences.
  • Project experience preferences live under .forgeflow/<project>/ and capture UI, product copy, accessibility, and project workflow preferences.

Safe Agent Injection

  • Context packs, memory context, and project intelligence can include compact user profile guidance only after the profile quality gate passes.
  • Warning or failed profile checks produce a gate note instead of raw profile text, keeping unusable or mis-scoped entries out of agent packets.
  • The guidance is explicitly advisory and never overrides current instructions, correctness, security, accessibility, validation evidence, or product judgment.

Runtime And Docs Coverage

  • New runtime helpers cover profile recording, checking, showing, and shared profile contracts.
  • Install, update, health, runtime-helper, release, and command-coverage checks include the profile surface.
  • README and wiki docs now explain storage locations, privacy boundaries, command usage, and how profile guidance reaches agents.
