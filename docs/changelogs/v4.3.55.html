Forgeflow V4.3.55 · 8 June 2026

Release: Operating Model Context Integration

Forgeflow 4.3.55 makes the project operating model usable during everyday workflows. The model now has a slash command surface and context packs inject compact advisory guidance into agent packets without rebuilding project intelligence recursively.

Project Model Command

  • /forgeflow-project-model builds and displays the local advisory project operating model.
  • The command supports --json, --refresh, and --out <path> with safe argument forwarding.
  • Health and update command inventory now includes the project-model command.

Context Pack Injection

  • Context packs write project-operating-model.md beside latest packet artifacts.
  • Agent packets include read-first, avoid-first, validate-first, high-care file, proof-boundary, and review-policy hints.
  • Synthesis metadata and agent context contracts mark the model as verify-before-use guidance.

Safety Boundary

  • Context packs consume existing operating-model artifacts instead of rebuilding project intelligence during packet generation.
  • Missing model artifacts degrade to metadata-only skipped guidance instead of triggering recursive work.
  • Review-auto behavior remains unchanged while this prepares future sandbox policy work.
