Forgeflow V4.3.45 · 2 June 2026

Release: Safer Wrappers, Follow-Through Capture, And Release Consumption

Forgeflow 4.3.45 tightens the local command surface and makes release, first-run, and learning follow-through easier to complete without guessing the next capture step.

Release Consumption Rollup

  • Added /forgeflow-release-consumption for a compact consumed-or-attention summary after release follow-through.
  • Added render-release-consumption-rollup.js with read-only default behavior, explicit --save snapshot writing, and explicit --with-smoke downstream smoke execution.
  • Wired the new helper through install, update, health, release-check, README, and wiki surfaces so consumers receive it through normal updates.

First-Run And Learning Follow-Through

  • /forgeflow-first-run now preserves quoted multi-word project names while keeping unsafe shell characters rejected.
  • /forgeflow-first-run-simulator now emits a trial plan, record-after template, and explicit follow-up status so first-run trials do not lose outcome evidence.
  • /forgeflow-learning-action now includes capture guidance and runnable templates for review outcomes, next-work outcomes, agent feedback, first-run results, user profile gaps, and project learnings.

Context And Wrapper Safety

  • /forgeflow-context-advisor now includes review-wave automation guidance with separate preview, write, and build commands plus a clear stop rule.
  • Four low-risk command wrappers moved to regular-file helper probes and env-scrubbed Node execution.
  • Wrapper smoke tests now cover quoted first-run project names and reject leftover unsafe --project-name tokens.
