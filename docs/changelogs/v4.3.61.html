Forgeflow V4.3.61 · 8 June 2026

Release: Generated Architecture Docs

Forgeflow 4.3.61 lands Phase 8 of the architecture intelligence path. Forgeflow can now render advisory architecture documentation from existing local topology, project intelligence, operating-model, and learning artifacts.

Architecture Command

  • New /forgeflow-architecture renders a local architecture report with evidence sources, major domains, entrypoint hints, hotspots, high-care files, risk zones, validation norms, known gaps, and next refresh action.
  • New render-architecture-docs.js supports Markdown and JSON output for human review and local automation.
  • Default output is read-only; --write stores local .forgeflow/<project>/context/architecture.md and architecture.json only.

Evidence Boundaries

  • The report uses existing static code topology, project intelligence, project operating model, and project learnings.
  • Missing evidence is surfaced as a gap instead of a hard failure.
  • Invalid JSON, symlinked artifacts, and unsafe local artifact reads are reported without following unsafe files.
  • The report labels unresolved imports and skipped dynamic imports as static-analysis hints, not runtime proof.

Runtime And Docs

  • Install, update, health, runtime contract, release-check, release-gate, README, and wiki references now include the architecture helper and command.
  • Command coverage now tracks 95 Forgeflow commands.
  • Release readiness now checks 122 managed runtime helper sources.

Safety Boundaries

  • The command does not edit repository docs by default.
  • It does not execute project code, start servers, trace runtime behavior, call GitHub, spawn agents, commit, push, publish, or claim review approval.
