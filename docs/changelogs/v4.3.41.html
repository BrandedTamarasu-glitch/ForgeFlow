Forgeflow V4.3.41 · 2 June 2026

Release: Safer Workflow Helpers And Clearer Learning Prompts

Forgeflow 4.3.41 tightens the lifecycle command wrappers and makes profile, learning, runtime inventory, and validation guidance easier to act on without making optional helpers mandatory.

Lifecycle Command Safety

  • /consult, /implement, /plan, /research, and /review now use scrubbed Node helper calls in the touched executable snippets.
  • Optional memory helpers warn and continue when unavailable instead of blocking otherwise-valid workflows.
  • /review keeps its inline routing fallback when the shared route helper is missing.

Profile And Learning Guidance

  • /forgeflow-profile-bootstrap now reports setup readiness and the next missing profile prompt for operating preferences and project style inputs.
  • /forgeflow-learnings status output now includes workflow-ending capture prompts for review, next-work, and agent-feedback evidence.
  • All capture guidance remains explicit and user-driven; Forgeflow does not infer or auto-record preferences or outcomes.

Runtime And Validation Refinements

  • Runtime inventory now groups command-wrapper contract and batch helpers under the shared command-wrapper surface.
  • Validation plans now expose a single first_failure_action before the full failed-command list.
  • README and wiki docs describe the new setup readiness, workflow-ending capture, runtime grouping, and validation-plan behavior.

Validation

  • Full scripts/forgeflow/test-*.js suite
  • node scripts/forgeflow/smoke-check.js --mode source --json
  • node scripts/forgeflow/test-release-version.js
  • git diff --check
  • Forgeflow full-mode review approved after Smith and Warden findings were fixed
