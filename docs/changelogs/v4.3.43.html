Forgeflow V4.3.43 · 2 June 2026

Release: Context Advisor, Runtime First-Use, And Release Consumption

Forgeflow 4.3.43 turns several advisory helpers into clearer command-surface guidance: context budget advice is now directly runnable, telemetry quality uses consistent trust language, first-use paths match the active runtime, and release follow-through now says whether a release appears consumed.

Context And Wrapper Safety

  • New /forgeflow-context-advisor wraps advise-context.js with safe argument handling for budget, savings, topology coverage, trend deltas, and proof-preserving trim guidance.
  • The command is installed through the managed command inventory, covered by health checks, and documented in README and wiki user paths.
  • The next wrapper cleanup batch reduced command-wrapper contract drift from 40 issues to 32 while preserving helper fallback behavior.

Telemetry And First-Use Guidance

  • /forgeflow-telemetry-quality now reports trusted sources, weakest sources, confidence, and one next quality action using the same vocabulary as learning status.
  • /forgeflow-first-useful-win now supports --runtime claude-code|codex and prints slash commands or source-helper commands that match the selected runtime.
  • The first-use next action is copy-pastable in both evidence-present and no-signal states.

Release Follow-Through

  • /forgeflow-release-follow-through now includes a release-consumption verdict with consumed state, confidence, blockers, informational follow-ups, and the next verification command.
  • Install attention remains blocking for real install drift, while mode-only informational drift stays informational.
  • Release and first-use tests cover the new verdict and runtime command paths.

  • node scripts/forgeflow/test-release-version.js
