Forgeflow V4.3.64 · 9 June 2026

Release: Architecture Intelligence Context Injection

Forgeflow 4.3.64 lands Phase 11 of the architecture intelligence path. Context packs now inject compact architecture, ownership, and invocation guidance into agent packets when local artifacts are present.

Context Pack Guidance

  • build-context-pack.js now writes architecture-intelligence.md alongside the existing packet artifacts.
  • Agent packets include a compact Architecture Intelligence section with read-first, high-care owner-surface, invocation, validate-first, and gap hints.
  • Synthesis metadata records architecture, ownership, and invocation artifact status so downstream review can see whether guidance was present, missing, or invalid.

Agent Safety Contract

  • Agent context contracts now list architecture-intelligence as verify-before-use advisory evidence.
  • The packet boundary explicitly says these hints are static architecture, ownership, and invocation evidence only.
  • Agents must still verify current files, tests, runtime behavior, and review artifacts before acting.

Token Discipline

  • Forgeflow injects a compact derived summary instead of raw architecture, ownership, and invocation JSON.
  • Missing artifacts become metadata gaps rather than hard failures.
  • Existing context budget, output-contract, and release-readiness checks cover the new block.
