Forgeflow V4.3.20 · 27 May 2026

Release: Sharper Local Intelligence And Release Confidence

Forgeflow 4.3.20 improves how local project signals turn into safe next actions. It adds clearer context-budget trimming guidance, first-run failure-digest handling, installed-runtime post-release checks, support-bundle redaction previews, and stronger project-intelligence action cards.

Context And Failure Guidance

  • Context-budget warnings now include advisory trim plans with target compact tokens, reduce-by estimates, focused-packet commands, and a stop rule for raw-required evidence.
  • Failure-digest reporting now distinguishes normal first-run missing state from stale or invalid digest state.
  • Trends and reports point users at the first digest capture path without implying that a missing first digest is already a failure.

Release And Support Safety

  • Post-publish release readiness now checks installed-runtime dogfood evidence alongside plugin version, tag, changelog, release notes, source smoke, and update smoke.
  • Support bundles now include a snippet-free redaction preview with sensitive-category counts before sharing.
  • The redaction preview flags private URLs, assignment secrets, local paths, relative paths, and repo-relative file references without printing matched values.

Project Intelligence Action Cards

  • Project-intelligence next-work candidates now include evidence strength, what-to-change, how-to-prove, stop-when, start, validation, and proof-boundary fields.
  • Context-advisor, import-gap, project-learning, freshness, feedback, and review-outcome candidates now use source-specific proof commands instead of unrelated refresh commands.
  • Stale agent feedback is downgraded to weak evidence with a stale-specific stop condition, keeping local learning signals advisory until current-code proof exists.
