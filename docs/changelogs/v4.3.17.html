Forgeflow V4.3.17 · 27 May 2026

Release: Smoke Gate Import-Gap Triage Fix

Forgeflow 4.3.17 fixes a downstream smoke false positive where expected production-scope import gaps, such as JSON or asset imports, could still warn even after code-map triage marked them as expected.

Smoke Check

  • /forgeflow-smoke now gates the code-map check on needs_review_total instead of all production-scope import gaps.
  • Expected production-scope gaps still appear in the code-map details, but no longer cause a warning when triage says zero gaps need review.
  • The warning text now distinguishes gaps that need review from total production-scope gaps.

Regression Coverage

  • The smoke test fixture now includes a production JSON import that is correctly triaged as expected.
  • The regression asserts that code-map smoke passes when production_total is nonzero and needs_review_total is zero.
