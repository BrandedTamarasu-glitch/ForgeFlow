Forgeflow V4.3.36 · 29 May 2026

Release: Review Waves, Outcome Evidence, And Failure Capture

Forgeflow 4.3.36 turns the latest efficiency work into a more complete operating path: smaller review waves, better outcome evidence prompts, safer failed-validation capture, and explicit profile bootstrap prompts.

Review Budget Workflow

  • New /forgeflow-review-wave-prep converts over-budget context telemetry into the first focused review-wave command.
  • /forgeflow-context-wave-plan --write-wave-files can now produce explicit file-list inputs for follow-up packet builds.
  • Review-wave prep stays advisory: it does not rebuild packets, spawn reviewers, edit source files, commit, or push.

Evidence Calibration

  • New /forgeflow-next-work-ranking ranks next work from current local evidence, confidence, demotion conditions, and validation hints.
  • New /forgeflow-outcome-capture-plan shows missing next-work, review, and agent-feedback outcome streams with concrete recorder prompts.
  • A shared runtime-inventory.js now backs command and runtime-helper inventory parity checks.

Profile And Failure Capture

  • New /forgeflow-profile-bootstrap previews explicit operating and project experience preferences, supports --prompts, and writes only with --write.
  • New /forgeflow-validation-failure-capture maps failed validation commands to safe capture modes and failure-digest paths without executing commands.
  • Command-output compaction now includes a safe build mode while preserving exact diffs, patches, hashes, and file lists raw.

Validation

  • Full scripts/forgeflow/test-*.js suite
  • node scripts/forgeflow/smoke-check.js --mode source --json
  • git diff --check
  • Install, update, command coverage, runtime inventory, release-version, docs, and wrapper-safety guards
