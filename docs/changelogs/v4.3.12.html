Forgeflow V4.3.12 · 26 May 2026

Release: Guided Repair And Learning Refinements

Forgeflow 4.3.12 packages the five-phase refinement track: shared public-safe boundaries, evidence-based release-note drafting, explicit project-readiness state, reviewer-feedback quality rollups, and a first-run guided repair command.

Public-Safe Output

  • privacy-boundary.js centralizes sensitive-content checks used by learning, adoption, feedback, and release helpers.
  • Private URLs, hostnames, local paths, secrets, and unsafe free-form categories are filtered consistently before public-facing summaries are rendered.
  • Existing evidence and adoption helpers now share the same privacy contract instead of carrying separate pattern checks.

Release And Readiness

  • render-release-notes.js drafts public-safe release-note material from local commit, changelog, manifest, and validation evidence.
  • build-project-intelligence.js now reports an explicit readiness state: ready, needs-refresh, needs-triage, or blocked.
  • Project intelligence keeps top risks uncapped in JSON while rendering a compact Markdown summary for review prep.

Reviewer Feedback

  • rollup-agent-feedback.js summarizes local reviewer feedback by agent, signal, promotable count, corrective count, skipped invalid/private lines, and safe advisory examples.
  • Feedback remains advisory-only and still requires confidence, evidence, and current-code verification before becoming project-learning guidance.

First-Run Guided Repair

  • /forgeflow-repair prints a non-mutating repair plan with offline version status, installed inventory checks, health failures, repair commands, and manual settings guidance.
  • The repair plan defaults to the local ~/.claude install, never edits settings.json, and separates manual actions from copy-pasteable commands.
  • Follow-through now includes /forgeflow-health, restart guidance for command/hook reloads, and an explicit downstream /forgeflow-smoke step.

Validation

  • Focused tests were added for each helper and wired into the release gate where applicable.
  • The full local scripts/forgeflow/test-*.js suite passed before tagging.
  • Forgeflow review approved all five phases, with final Arbiter and Compass approval for the guided repair slice.
