Forgeflow V4.3.68 · 9 June 2026

Release: Project Readiness Dashboard Panel

Forgeflow 4.3.68 lands Phase 15 of the dashboard and operating-model track. The optional local dashboard now shows current project readiness directly in the UI instead of requiring users to inspect several command outputs.

Dashboard Panel

  • The Project Readiness panel consumes GET /api/readiness and renders status cards for project health, learning status, context budget, release readiness, dogfood report state, and refresh-plan readiness.
  • The panel shows the current next action as a copy-only command so users can decide when to run it in their terminal.
  • Status is visible as text and badge labels, not color alone.

Responsive UI

  • The dashboard now stacks cleanly on narrow viewports instead of clipping the right-side panel.
  • The readiness card layout keeps long summaries and command text inside their containers.
  • The panel stays dense and operational rather than becoming a marketing surface.

Safety Boundary

  • The UI only reads the readiness API.
  • It does not run commands, refresh artifacts, write files, call GitHub, spawn agents, export telemetry, or promote automation.
  • The copy button copies the suggested command text only.

Validation

  • node --test services/dashboard/__tests__/readiness.test.js
  • Desktop and mobile Playwright screenshot smoke checks
  • Full scripts/forgeflow/test-*.js suite
  • node scripts/forgeflow/render-release-readiness.js --json
