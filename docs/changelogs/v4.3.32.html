Forgeflow V4.3.32 · 29 May 2026

Release: Guided Adoption And Review Safety

Forgeflow 4.3.32 turns recent learning and adoption work into clearer user-facing checks. It adds early-value reporting, safer review-auto previews, a next-action contract audit, profile resolution options, and insight-injection diffs.

Adoption And Learning

  • New /forgeflow-first-useful-win summarizes public-safe early wins from first-run results, pilot evidence, agent feedback, and learning status.
  • Profile review now renders explicit accept, reject, supersede, and defer choices so user preference updates stay confirmed and reversible.
  • README, workflow docs, user paths, and context-intelligence docs now describe the updated adoption and profile-resolution flow.

Context And Next Actions

  • /forgeflow-insight-injection can compare latest packet artifact decisions with a baseline manifest.
  • Missing core packet artifacts now point users back to build-context-pack.js --json instead of a generic contract audit.
  • New /forgeflow-next-action-audit spot-checks representative helper next actions so copy-pastable commands stay separate from explanatory reasons.

Review-Auto Preview

  • New /forgeflow-review-auto-classify --findings <json> previews safe, risky, and blocker buckets without editing files or dispatching workers.
  • The classifier follows the published /review-auto policy: Warden and dependency findings are surfaced as risky, while migrations and secrets remain blockers.
  • Command wrappers, health inventory, install manifest, update tests, and release gates now cover the new helpers.

Validation

  • Full scripts/forgeflow/test-*.js suite
  • node scripts/forgeflow/smoke-check.js --mode source --json
  • git diff --check
  • Focused first-useful-win, insight-injection, profile-review, next-action-contract, review-auto-classifier, install, update, docs, and runtime-helper tests
  • Forgeflow review revisions from Smith, Compass, and Atlas were addressed before release
