Forgeflow V4.3.2 · 21 May 2026

Patch Release: Update Flow Auto-Repair

Forgeflow 4.3.2 closes the gap found during real downstream install verification: an install could be on the latest SHA while still missing required managed runtime helpers.

Update Flow

  • Plain /update-forgeflow now checks required managed files even when ~/.claude/forgeflow-version already matches the latest upstream SHA.
  • If required managed files are missing, the updater automatically runs the repair sync instead of returning up-to-date.
  • The updater reports the missing-file repair path explicitly with Missing managed files detected (...); running repair sync.
  • Explicit /update-forgeflow --repair remains available for corrupted managed files or manual recovery.

Docs

  • Updated /update-forgeflow command docs so the fallback contract matches the script-backed updater behavior.
  • Updated Settings And Recovery to explain that plain update now auto-repairs missing required managed files.

Validation

  • test-update-forgeflow.js covers latest-SHA incomplete installs.
  • Install, health, version, docs, release-version, and diff checks passed locally.
