Forgeflow V4.3.31 · 28 May 2026

Release: Insight Injection And Safer Automation

Forgeflow 4.3.31 makes agent guidance more explainable for users. It adds a read-only insight-injection view, tightens first-run orientation, surfaces project-map evolution, and narrows review-auto automation guardrails.

Insight Visibility

  • New /forgeflow-insight-injection shows packet artifact decisions, per-agent signal contracts, quality-gate controls, and copy-pastable next commands.
  • The helper reads generated agent packets and reports whether each insight section is present per agent.
  • Profile review now shows injection eligibility, issue counts, and safe next steps before agent-heavy work.

First-Run And Project Map

  • /forgeflow-first-run now points new users through project-map evolution and insight-injection inspection.
  • /forgeflow-health-timeline now includes project-map evolution with command-only next actions and separate reasons.
  • Command docs, README, and wiki pages now describe the updated onboarding and context-inspection path.

Review-Auto Guardrails

  • /review-auto now documents a conservative auto-fix eligibility matrix.
  • Same-file auto-fix findings must be serialized, with file snapshots refreshed between workers.
  • Risky, security-sensitive, multi-file, dependency, migration, secret, and product-judgment findings remain human-review items.

Validation

  • Full scripts/forgeflow/test-*.js suite
  • node scripts/forgeflow/smoke-check.js --mode source --json
  • git diff --check
  • Focused insight-injection, first-run, project-health-timeline, command-coverage, release-version, docs, install, update, and runtime-helper tests
  • Forgeflow review approval from Smith, Warden, Lumen, and Atlas
