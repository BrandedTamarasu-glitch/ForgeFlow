Forgeflow V4.3.33 · 29 May 2026

Release: Update Confidence And First-Task Evidence

Forgeflow 4.3.33 adds five refinement features that make update health, review-auto evidence, learning-signal trust, first-task outcomes, and helper output contracts easier to verify.

Install And Output Confidence

  • New /forgeflow-update-verify checks installed version state and runtime drift after update or repair, then reports ready, restart, or repair guidance.
  • New /forgeflow-output-contract spot-checks representative helper output for status, next command, reason, and advisory boundary fields.
  • Update verification now passes an explicit source root, avoids false repair reports from unrelated directories, and keeps restart guidance in next_reason.

Review And Learning Evidence

  • New /forgeflow-review-auto-evidence --findings <json> writes a local classification artifact for safe, risky, and blocker buckets.
  • Review-auto evidence writes are confined to the project Forgeflow directory.
  • New /forgeflow-learning-policy shows or seeds configurable learning-signal decay rules consumed by /forgeflow-learning-status.

First Real Task

  • New /forgeflow-first-task-report summarizes first real work-item success signals, blockers, evidence counts, and next adoption action.
  • The report keeps raw project records out of the rendered output while still showing useful aggregate signals.
  • README, workflow docs, user paths, context-intelligence docs, health inventory, install manifest, update tests, and release gates cover the new surfaces.

Validation

  • Full scripts/forgeflow/test-*.js suite
  • node scripts/forgeflow/smoke-check.js --mode source --json
  • git diff --check
  • Install, update, command coverage, runtime-helper, release-version, docs, and wrapper-safety guards
  • Forgeflow final synthesis review approved after Smith, Warden, Compass, and Arbiter revisions were addressed
