Forgeflow V4.3.22 · 27 May 2026

Release: Profile Refinement And First-Run Guidance

Forgeflow 4.3.22 refines the local profile system and adds a compact first-run path for new users. The release keeps profile learning advisory, makes conflicts visible, and improves release verification summaries.

Profile Refinement

  • User profile checks now surface advisory suggested updates without writing inferred preferences automatically.
  • Potentially conflicting active preferences now render the competing entries, the next command, and follow-up guidance.
  • Placeholder profile text is rejected while normal guidance phrasing remains valid.
  • Role-specific guidance explains how Atlas, Compass, Lumen, Warden, Smith, and Arbiter should use profile preferences.

First-Run Path

  • New /forgeflow-first-run command prints a compact net-new user path for install verification, project orientation, profile readiness, and one bounded work item.
  • The first-run helper is installed with the runtime helpers and covered by release, update, health, command, and install checks.
  • Quick Start, package onboarding, template installer, User Paths, hosted docs, and README now point users at the first-run guide.

Release Verification

  • Post-publish release readiness now includes a concise shareable summary, passed evidence list, and attention evidence list.
  • /forgeflow-release-check explicitly changes to the repository root before running relative release-check commands.
