Forgeflow V4.3.9 · 22 May 2026

Release: First-Run Adoption And Project Intelligence

Forgeflow 4.3.9 focuses on helping a new user understand whether Forgeflow is worth using, then carrying project learning forward across work items.

New User And Adoption Flow

  • /forgeflow-pilot --path new-user prints a first-real-task trial path with readiness, project orientation, implementation, review, and decision steps.
  • /forgeflow-adoption prints a concise adoption pack with fit guidance, proof boundaries, first-trial steps, existing pilot evidence, and repeat, expand, fix, or defer criteria.
  • The adoption pack summarizes local pilot-evidence rollups when available without exposing raw project artifacts.

Learning And Guidance

  • Quality gate recommendations now include evidence and clear conditions so users know why a warning appeared and what resolves it.
  • build-project-intelligence.js synthesizes trends, code topology, project learnings, validation patterns, risks, and recommended next actions into local rollup artifacts.
  • record-agent-feedback.js captures local feedback on useful, unclear, ignored, or incorrect agent guidance with explicit promotion rules and privacy guards.

Validation

  • Release checks now cover the adoption pack, project intelligence rollup, agent feedback recorder, and enhanced guidance contract.
  • Full local release-gate validation passed before tagging.
  • Specialist review approved the adoption flow and feedback recorder after command-safety and privacy-boundary fixes.
