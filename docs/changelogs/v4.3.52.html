Forgeflow V4.3.52 · 4 June 2026

Release: Smoke And Runtime Refinements

Forgeflow 4.3.52 tightens failure-digest command forwarding, reduces runtime inventory pressure, and keeps downstream smoke checks budget-safe during dirty-tree work.

Failure Digest Command Parity

  • /forgeflow-failure-digest now forwards the documented --file, --out, and --json arguments to the runtime helper.
  • The failure-digest test now exercises the real CLI path with file input, explicit output, and JSON reporting.
  • The helper still preserves raw-required output for unsafe command classes.

Runtime Inventory Pressure

  • runtime-inventory.js now separates learning recorders, learning evidence, user profile helpers, and adoption guidance helpers.
  • The largest helper group dropped below the attention threshold, moving runtime coordination pressure from attention to watch.
  • Runtime parity checks still compare command coverage, health inventory, release gates, and install manifests.

Budget-Safe Smoke

  • Project-learning smoke now builds a thin context packet, which is enough to verify packet generation and latest-insight injection without overbuilding review context.
  • Downstream smoke now stays green for small dirty-tree refinements instead of tripping a full-context budget warning.
  • The higher-risk commands/review.md safe-args work remains paused for a separate design/review slice.
