Forgeflow V4.3.66 · 9 June 2026

Release: Dogfood Evidence Refresh Plan

Forgeflow 4.3.66 lands Phase 13 of the architecture intelligence path. The new refresh plan turns an incomplete dogfood report into ordered local commands so users can gather the missing evidence before reconsidering higher automation.

Refresh Planning

  • /forgeflow-dogfood-refresh-plan reads the dogfood report state and lists the next local evidence commands in order.
  • render-dogfood-refresh-plan.js handles missing architecture, ownership, invocation, topology, and context-pack synthesis evidence.
  • The planner points users back to /forgeflow-dogfood-report --write after the missing evidence steps are complete.

Safety Boundary

  • The helper is read-only and does not run refresh commands.
  • It does not write artifacts, spawn agents, edit files, commit, push, call GitHub, or promote automation.
  • Invalid artifacts are surfaced as repair-first evidence instead of being treated as readiness.

Install And Release Coverage

  • The command and runtime helper are managed by install, update, health, runtime inventory, and release readiness.
  • README and wiki pages now document the refresh plan alongside the dogfood report.
  • Release gate docs include the refresh-plan fixture test so command and helper drift is caught before shipping.
