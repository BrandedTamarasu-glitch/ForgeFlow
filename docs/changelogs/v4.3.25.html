Forgeflow V4.3.25 · 28 May 2026

Release: Learning Review And Runtime Drift Checks

Forgeflow 4.3.25 makes local learning signals easier to act on and adds explicit installed-runtime drift checks. The release keeps project and pattern memory advisory, reviewable, and safe for cross-project use.

Learning Status UX

  • /forgeflow-learning-status now groups recommendations into Fix First, Watch, and Healthy lanes.
  • Next-work outcome history now records confidence calibration bands so Forgeflow can learn whether its recommendations were useful, corrective, or noisy.
  • Artifact contracts and tests cover the new grouped status and confidence calibration fields.

Pattern And Runtime Review

  • New /forgeflow-pattern-review renders dry-run pattern promotion candidates with a redaction checklist and a manual promotion boundary.
  • New /forgeflow-runtime-drift compares source runtime helpers with installed helpers and reports missing files, content drift, mode drift, and syntax failures.
  • Health, release-check, update, install-manifest, and installed-runtime dogfood coverage now include both new helpers.

Release Guidance

  • Release docs now explain that sandboxed /forgeflow-release-verify --github runs may report network-unavailable without proving the release or tag is missing.
  • README and wiki docs describe the new learning review, pattern review, and runtime drift workflows.
  • The new commands remain local-first: they report evidence and recommendations without auto-promoting patterns or repairing installed files.

Validation

  • Full scripts/forgeflow/test-*.js suite
  • node scripts/forgeflow/smoke-check.js --mode source --json
  • git diff --check
  • Focused command safety, command coverage, pattern review, runtime drift, learning status, and next-work outcome tests
