Forgeflow V4.3.51 · 3 June 2026

Release: Efficiency Guardrails

Forgeflow 4.3.51 tightens the safe automation path for context waves, command-output reduction, runtime inventory, and downstream dogfood loops.

Context Wave Proof

  • /forgeflow-context-wave-plan now reports per-wave budget status, proof contracts, and a copy-ready verification command.
  • The verification handoff uses the same runnable node scripts/forgeflow/check-context-budget.js --root .forgeflow --warn-only --json form used by focused wave builds.
  • Wave planning still writes files only when --write-wave-files is explicit and does not spawn reviewers or trim proof files.

Safe Output Presets

  • compact-command-output.js now supports --preset auto for common test, typecheck, lint, build, log, search, status, tree, and JSON commands.
  • Exact-output commands such as diffs, SHAs, patches, file-list flags, and find remain raw-required.
  • /forgeflow-capture-output and /forgeflow-failure-digest now accept preset arguments and report preset metadata.

Runtime And Dogfood Guidance

  • runtime-inventory.js now exposes canonical consolidation checks with coordination-pressure guidance.
  • /forgeflow-release-consumption-loop now includes a downstream efficiency trial checklist for context-wave pressure, output pressure, runtime registry pressure, and learning capture.
  • README, command docs, and wiki pages were updated for the new preset, proof, registry, and dogfood surfaces.

Validation

  • Full scripts/forgeflow/test-*.js suite
  • node scripts/forgeflow/smoke-check.js --mode source --json
  • node scripts/forgeflow/render-release-readiness.js --json
  • Forgeflow review approved by Smith, Warden, Lumen, and Atlas after the verification-command, raw-required find, CLI preset, and explicit-preset findings were fixed.
