Forgeflow V4.3.48 · 3 June 2026

Release: Workflow Readiness Queue

Forgeflow 4.3.48 adds a read-only workflow-readiness queue that turns the next five refinement areas into one actionable status surface while keeping high-risk review command work paused.

Workflow Readiness

  • Added /forgeflow-workflow-readiness to show the next safe action across review waves, outcome calibration, profile setup, telemetry quality, and runtime inventory parity.
  • The helper reports five phase records with status, next command, reason, validation commands, advisory boundary, and local evidence.
  • High-risk commands/review.md safe-args work is listed separately as paused instead of being routed automatically.

Safety And Runtime Wiring

  • The new command accepts only --json and does not write wave files, record outcomes, infer preferences, change routing, repair installs, commit, push, or call GitHub.
  • Incomplete context now routes to /forgeflow-context-advisor --record instead of /review.
  • Partial helper output shapes degrade safely, including missing outcome streams and missing telemetry trust summaries.
  • Managed runtime, health inventory, update, command coverage, README, and wiki docs are updated for the new helper.
