Forgeflow V4.3.5 · 21 May 2026

Patch Release: Sharper Code-Map Resolution

Forgeflow 4.3.5 fixes the first false-positive class found by the GSD-CIC pilot after import-gap triage shipped. Code topology now resolves suffix-style TypeScript modules and extensionless TSX re-exports before asking maintainers or agents to inspect them.

Resolver Fixes

  • Static code maps now resolve source suffix imports such as ./types/audit.types to matching files like audit.types.ts.
  • Extensionless TSX re-exports like ./password-icon now resolve to password-icon.tsx instead of surfacing as local missing modules.
  • Sensitive filename filtering still protects non-source artifacts, while source files with names such as password-icon.tsx remain part of the topology graph.

Downstream Validation

  • Verified on GSD-CIC after /update-forgeflow pulled commit 58df312.
  • Production-scope import gaps dropped from 40 to 26.
  • Needs-review import gaps dropped from 23 to 5; the remaining review items are dynamic local or alias imports.
  • /forgeflow-smoke completed with expected warnings only for the remaining import-gap review signal.

GitHub Issue

This release addresses issue #2: code-map import resolver false positives for .types-suffixed specifiers and extensionless .tsx re-exports.
