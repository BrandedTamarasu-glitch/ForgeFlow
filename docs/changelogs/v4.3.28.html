Forgeflow V4.3.28 · 28 May 2026

Release: Update And Guidance Signal Refinements

Forgeflow 4.3.28 smooths the update, release, context-budget, and profile-review loops. It improves repair behavior for newly added runtime helpers, separates actionable drift from mode-only drift, trims compact topology output, and quiets first-run failure-digest noise.

Update And Runtime Confidence

  • /update-forgeflow --repair now recognizes any managed non-test helper under scripts/forgeflow/, so newly added helpers can be installed in one repair pass.
  • /forgeflow-runtime-drift now reports actionable drift and mode-only drift separately.
  • Release verification treats mode-only helper drift as informational while keeping missing helpers, content drift, and syntax failures actionable.

Context And Guidance Signals

  • Compact code-topology output now caps lower-value node, edge, and neighborhood detail while preserving hotspots, changed sections, and review-useful neighbors.
  • Missing first-run failure digests are informational until a failed command is captured, instead of becoming a next-work recommendation.
  • Profile review now includes explicit accept/supersede commands and a resolution flow for confirmed user preferences.

Validation

  • Full scripts/forgeflow/test-*.js suite
  • node scripts/forgeflow/smoke-check.js --mode source --json
  • git diff --check
  • Focused update, install manifest, runtime drift, release verify, topology, trends, report, profile review, docs, and release-version tests
